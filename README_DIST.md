# YandexMUSIC_ANALYTICS

Аналитический пайплайн обработки датасета Yambda-500M от Яндекса.
Анализ 500 миллионов событий взаимодействий пользователей с музыкальным сервисом.

## Быстрый старт

### 1. Установка

Двойной клик по `setup.bat`. Откроется консоль с выбором:

- **[1] Демо-режим** — 180 МБ места, оффлайн, готово за 2-3 минуты
- **[2] Полный режим** — 15 ГБ скачивания, нужен интернет, готово за ~1 час

Для быстрого знакомства выбирайте демо-режим.

### 2. Запуск анализа

Двойной клик по `run.bat`. Запустятся все 7 аналитических задач.
Результаты сохранятся в `data/results/` как PNG-графики.

## Состав дистрибутива

| Файл / папка | Назначение |
|---|---|
| `setup.bat` | Установщик: распакует Python, поставит зависимости |
| `run.bat` | Запуск всех 7 задач |
| `src/` | Код проекта |
| `scripts/` | Скрипты (download_data.py, prepare_db.py, run_all.py) |
| `requirements.txt` | Список Python-зависимостей |
| `data_demo.zip` | Демо-данные (5000 пользователей) |
| `python_embed.zip` | Портативный Python 3.11 (Windows) |

После установки появятся:
- `python/` — распакованный Python с зависимостями
- `data/demo/` (демо-режим) или `data/raw/` (полный режим)
- `data/processed/` — служебные кеши
- `data/results/` — итоговые графики

## Системные требования

- Windows 10/11 (64-bit)
- 8 ГБ RAM минимум (16 ГБ для полного режима)
- Свободное место на диске:
  - Демо: ~500 МБ
  - Полный: ~30 ГБ

## Аналитические задачи

1. Динамика прослушиваний по месяцам
2. Распределение сессионной активности
3. Граф ко-лайков и музыкальные сообщества (Louvain)
4. Энтропия Шеннона разнообразия слушания
5. Тепловая карта по часам и дням недели
6. Граф ко-прослушиваний + UMAP-валидация (только в полном режиме)
7. Прогнозирование оттока пользователей (Random Forest + LR)

## Архитектура БД

Используется DuckDB — встраиваемая аналитическая СУБД, оптимизированная под
колоночные агрегации на больших объёмах. Развёртывание автоматическое:
`pip install duckdb` плюс распаковка `analytics.duckdb` из дистрибутива.
Подключение в коде без сетевых параметров (embedded-архитектура, как SQLite
для аналитики).

## Технологический стек

- Python 3.11
- Polars (табличная обработка)
- DuckDB (аналитическая БД)
- NetworkX + python-louvain (графы)
- UMAP-learn (нелинейное снижение размерности)
- scikit-learn (модели)
- Matplotlib + Seaborn + adjustText (визуализация)
